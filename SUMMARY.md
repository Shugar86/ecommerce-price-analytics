# SUMMARY - Краткое резюме проекта

## ✅ ЧТО РЕАЛИЗОВАНО

### 🌐 Веб и ИИ (ВКР / практика)
- **Веб-приложение аналитика:** FastAPI + Jinja2, порт **8000** (`app/web/`) — дашборд, поиск, графики истории цен, аномалии, **кандидаты** TF‑IDF (ревью в UI), метрики полноты полей и exact-пересечений ключей, CSV.
- **Воркер аналитики:** `app/ai_worker.py` + `app/ml/` — аномалии, TF‑IDF-кандидаты EKF↔TDM (порог `AI_MATCH_MIN_SCORE`, one-to-one), линейный прогноз; `app/quality/coverage.py` — агрегаты для дашборда. Таблицы `price_history`, `price_anomalies`, `product_matches` (с `match_kind` / `match_status`), `price_forecasts`.
- **Текст ВКР/практики:** **[VKR_AND_PRACTICE_REPORT.md](VKR_AND_PRACTICE_REPORT.md)** (главы 1–3). Честные формулировки по сопоставлению: **[docs/PRODUCT_SCOPE.md](docs/PRODUCT_SCOPE.md)**.

### 🏗️ Архитектура
- **6+ Docker-сервисов:** PostgreSQL, Adminer, Collector (ETL), **Web**, **AI worker**, Telegram Bot
- **Полная изоляция:** Каждый компонент работает независимо
- **Отказоустойчивость:** Падение одного сервиса не влияет на другие

### 📊 Модели данных
- **ExchangeRate:** Хранение курсов валют (USD → RUB)
- **Product:** Информация о товарах с денормализованными ценами
- **SQLAlchemy 2.0:** Современные type hints, Mapped типы

### 🔄 ETL-процесс (Collector)
- **ЦБ РФ API:** Парсинг XML (windows-1251) для курсов валют
- **FakeStore API:** REST API для зарубежных товаров (JSON)
- **TBM Market:** Потоковый парсинг YML (demo Big Data обработки)
- **UPSERT логика:** Избегание дубликатов через ON CONFLICT
- **Автоматизация:** Цикл обновления каждый час

### 🤖 Telegram-бот (Aiogram 3.x)
- `/start` — Приветствие и справка
- `/stats` — Статистика по товарам и курсам
- `/find <запрос>` — Поиск товаров с ценами
- **Асинхронность:** Обработка множества пользователей

## 🔥 КЛЮЧЕВЫЕ ФИШКИ РЕАЛИЗАЦИИ

### 1. Потоковый парсинг (Streaming)
```python
# Обработка больших XML без загрузки в память
context = etree.iterparse(stream, events=('end',), tag='offer')
for event, elem in context:
    process(elem)
    elem.clear()  # Освобождение памяти
```

### 2. UPSERT через SQLAlchemy
```python
# Атомарная операция INSERT ... ON CONFLICT UPDATE
stmt = insert(Product).values(...).on_conflict_do_update(
    index_elements=['external_id'],
    set_={'price': new_price}
)
```

### 3. Type Hints + SQLAlchemy 2.0
```python
# Современный синтаксис с полной типизацией
class Product(Base):
    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(500))
```

### 4. Docker Multi-Container
```yaml
# Единый Dockerfile, разные команды запуска
collector:
  command: python -m app.collector
bot:
  command: python -m app.bot
```

## 📞 ПОДДЕРЖКА

### Если что-то не работает

1. **Прочитать:** FAQ.md — 90% проблем решаются там
2. **Проверить:** `docker-compose logs <service>`
3. **Запустить:** `python check_system.py`
4. **Пересобрать:** `docker-compose down && docker-compose up -d --build`

### Типичные проблемы

| Проблема | Решение |
|----------|---------|
| Бот не отвечает | Проверить токен в .env |
| Нет данных | Подождать 1-2 минуты после запуска |
| Порт занят | Изменить порты в docker-compose.yml |
| БД недоступна | `docker-compose restart db` |
