# Как участвовать

Спасибо, что хотите помочь. Проект открыт для идей, багфиксов и улучшений — но придерживается строгой дисциплины: минимальный diff, точные формулировки и рабочие команды.

## Быстрый процесс

1. Форкните репозиторий.
2. Создайте ветку: `git checkout -b feature/краткое-описание`.
3. Сделайте изменения.
4. Запустите тесты и базовые проверки:
   ```bash
   python -m compileall app/ tools/ tests/
   pytest tests/ -q
   ```
5. Убедитесь, что `.env`, токены и сгенерированные артефакты не попали в коммит.
6. Откройте PR с описанием: что, зачем, как проверяли.

## Стиль кода

- Python 3.12+, type hints, docstrings для публичных функций.
- Имена переменных — понятные; сокращения только устоявшиеся (ETL, KPI, SKU).
- SQL-запросы читаемые; сложную логику выноси в именованные функции.
- Новые эндпоинты FastAPI — с явными статусами и обработкой ошибок.
- Миграции БД — только через Alembic.

## DoD для PR

- [ ] Код запускается и не ломает существующие тесты.
- [ ] Добавлены/обновлены тесты на новую логику, если применимо.
- [ ] Документация (`README.md`, `AGENTS.md`, `CHANGELOG.md`) обновлена при изменении контракта или запуска.
- [ ] В diff нет секретов, `.env`, `node_modules/`, `__pycache__/` и крупных бинарников.
- [ ] Сообщения коммитов — на русском или English, в формате `type(scope): описание`.

## Если сомневаетесь

Откройте issue и обсудите задачу до кодирования. Это экономит время всем.
