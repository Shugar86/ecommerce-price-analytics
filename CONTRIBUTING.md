# CONTRIBUTING — как участвовать в PriceDesk

Спасибо, что хотите помочь. PriceDesk открыт для идей, багфиксов и улучшений — но придерживается строгой дисциплины: минимальный diff, точные формулировки и рабочие команды.

## Быстрый процесс

1. **Форкните** репозиторий (или создайте ветку, если есть доступ).
2. **Создайте ветку** от актуального `main`:
   ```bash
   git checkout main
   git pull origin main
   git checkout -b feature/краткое-описание
   ```
3. **Внесите изменения.** Следуйте [AGENTS.md](./AGENTS.md): KISS, честность про границы, секреты только в `.env`.
4. **Проверьте код** локально:
   ```bash
   python -m compileall app/ tools/ tests/
   pytest tests/ -q
   ```
5. **Убедитесь, что в diff нет секретов**:
   ```bash
   git diff --stat
   git diff | grep -iE '(token|password|secret|github_pat|api_key|\.env)' || true
   ```
6. **Закоммитьте** в формате `type(scope): описание`:
   ```bash
   git commit -m "feat(matching): добавить порог по пересечению токенов"
   ```
7. **Откройте PR** с описанием:
   - Что изменилось и зачем.
   - Как проверяли (команды, тесты, скриншоты).
   - Риски и граничные случаи.

## Именование веток

| Префикс | Когда использовать |
|---------|--------------------|
| `feature/` | Новая функциональность |
| `fix/` | Исправление бага |
| `docs/` | Только документация |
| `refactor/` | Рефакторинг без изменения поведения |
| `test/` | Тесты |

## Стиль кода

- Python 3.11+, type hints, docstrings для публичных функций.
- Имена переменных — понятные; сокращения только устоявшиеся (`ETL`, `KPI`, `SKU`, `URL`).
- SQL-запросы читаемые; сложную логику выноси в именованные функции.
- Новые эндпоинты FastAPI — с явными статусами и обработкой ошибок.
- Миграции БД — только через Alembic.
- Коммит-сообщения — на русском или English, в формате `type(scope): описание`.

## DoD для PR

- [ ] Код запускается и не ломает существующие тесты.
- [ ] Добавлены/обновлены тесты на новую логику, если применимо.
- [ ] Документация (`README.md`, `AGENTS.md`, `CHANGELOG.md`) обновлена при изменении контракта или запуска.
- [ ] В diff нет секретов, `.env`, `node_modules/`, `__pycache__/`, крупных бинарников.
- [ ] Сообщения коммитов соответствуют соглашению.

## Если сомневаетесь

Откройте issue и обсудите задачу до кодирования. Это экономит время всем и защищает от архитектурных ловушек.
