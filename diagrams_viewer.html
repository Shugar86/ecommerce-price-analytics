Компоненты системы:
  • Collector — ETL-процесс сбора данных (работает по расписанию)
  • PostgreSQL — хранилище товаров и курсов валют
  • Bot — Telegram-интерфейс для пользователей

Логическая модель
erDiagram PRODUCTS { int id PK "Первичный ключ" varchar external_id "Уникальный внешний ID" varchar name "Название товара" varchar name_norm "Нормализованное название" float price_original "Оригинальная цена" varchar currency "Код валюты" float price_in_rub "Цена в рублях" varchar source_shop "Источник" varchar url "URL товара" varchar barcode "Штрихкод" varchar vendor_code "Артикул" varchar category_id "ID категории" datetime updated_at "Время обновления" } EXCHANGE_RATES { int id PK "Первичный ключ" varchar currency_code "Код валюты" float rate "Курс к рублю" datetime updated_at "Время обновления" } EXCHANGE_RATES ||--o{ PRODUCTS : "используется для конвертации"
Сущности:
  • PRODUCTS — таблица товаров (59K+ записей)
  • EXCHANGE_RATES — курсы валют от ЦБ РФ
  • Связь «многие-к-одному»: товары конвертируются по курсу валюты

⚡ Диаграмма деятельности

ETL-процесс
flowchart TD Start((Старт)) Start --> Init["Инициализация:\nподключение к БД"] Init --> FetchRates["Получить курсы валют\nот ЦБ РФ"] FetchRates --> Decision1{Успешно?} Decision1 -->|Да| SaveRates["Сохранить курсы в БД"] Decision1 -->|Нет| LogError1["Логировать ошибку"] SaveRates --> FetchProducts LogError1 --> FetchProducts FetchProducts["Сбор товаров\n(6 источников)"] FetchProducts --> Stream["Потоковое чтение\n(iterparse)"] Stream --> Parse["Парсинг элемента"] Parse --> Normalize["Нормализация"] Normalize --> Convert["Конвертация валюты"] Convert --> Upsert["Upsert в БД"] Upsert --> HasMore{Ещё товары?} HasMore -->|Да| Parse HasMore -->|Нет| Commit Commit["Commit транзакции"] Commit --> Sleep["Ожидание 1 час"] Sleep --> FetchRates style Start fill:#3fb950 style Commit fill:#58a6ff
Алгоритм ETL:
  • Цикл повторяется каждый час (UPDATE_INTERVAL = 3600)
  • Потоковый парсинг: lxml.etree.iterparse
  • Upsert: ON CONFLICT DO UPDATE (PostgreSQL)
